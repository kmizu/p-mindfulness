# mindfulness-supervisor

監督優先のマインドフルネスアプリ。瞑想の先生ではなく、安全モニター。

[MindfulAgents: A Multi-Agent Framework for Mindfulness Practice](https://arxiv.org/abs/2603.06926) で提案された3エージェント構成を参考に、リフレクションエージェント・パーソナライゼーションエージェント・エキスパートアラインメントエージェントを単一アプリとして実装しています。

## 概要

セッション前にカウンセラー風の会話型エージェントが現在の状態を確認します。回答を分析し、練習が助けになるか、害になるかを判断します。自己批判、内部監視、強制的な受け入れ、強迫的な継続、パフォーマンス思考などのパターンを検出し、長期記憶として蓄積していきます。

1. **会話チェックイン** — 2〜3ターンの対話（リフレクションエージェント）
2. **監督レビュー** — リスク評価・モードと時間の推奨（パーソナライゼーションエージェント＋長期記憶）
3. **セッション** — 短いガイド練習（30秒 / 1分 / 3分）、状態に合わせてカスタマイズ
   - 練習中は常に「これは悪化させている」ボタンが表示される
4. **振り返り** — 練習は助けになったか、プレッシャーを増やしたか
5. **履歴** — 過去のセッション一覧；セッション後に記憶を自動更新

## セットアップ

```bash
npm install
cp .env.example .env.local
# .env.local を編集してAPIキーを設定（どちらも任意）
npm run dev
```

http://localhost:3000（または /ja）をブラウザで開く。

## 環境変数

| 変数名 | 必須 | 説明 |
|---|---|---|
| `OPENAI_API_KEY` | 任意 | LLMエージェント（gpt-5.4）。未設定時はルールベース＋プリセット。 |
| `ELEVENLABS_API_KEY` | 任意 | 音声再生。未設定時はテキストのみ。 |
| `ELEVENLABS_VOICE_ID` | 任意 | 使用する音声ID。デフォルトは落ち着いた英語音声。 |

**APIキーなしでも完全に動作します。**

## アーキテクチャ

3つのエージェントが順番に実行されます：

- **リフレクションエージェント** — 会話型チェックイン（2〜3ターン）、`ReflectionProfile` を抽出
- **パーソナライゼーションエージェント** — プロフィール＋長期記憶＋ルールベース検出から `SessionPlan` を生成
- **エキスパートアラインメントエージェント** — プランから個人化されたガイダンステキストを生成

長期記憶（カウンセラー風ケースノート）はセッションごとに蓄積され、次回以降の推奨に反映されます。

## データ

セッションデータと長期記憶は `data/mindfulness.db`（SQLite）にローカル保存されます。
音声キャッシュは `data/tts-cache/` に保存されます。どちらもgitにはコミットされません。
